Tracking Changes Challenge

×

Objective

Learn how to track file changes and view commit history.

Instructions

  1. Modify an existing file (e.g., "index.html").
  2. Check the status of your repository using git status.
  3. View changes using git diff.
  4. Stage the modified file using git add.
  5. Commit the changes with a meaningful message.
  6. Check commit history using git log.

Branch Management Challenge

×

Objective

Create, switch, and merge branches to organize your work.

Instructions

  1. Create a new branch named "feature-branch".
  2. Switch to the new branch using git checkout.
  3. Modify or create a new file.
  4. Stage and commit the changes.
  5. Switch back to the main branch and merge the feature branch.
  6. Delete the feature branch after merging.

Merge Conflicts Challenge

×

Objective

Learn to identify and resolve merge conflicts effectively.

Instructions

  1. Create a new branch and modify a file.
  2. Switch to the main branch and modify the same file differently.
  3. Attempt to merge the branch, causing a conflict.
  4. Use git status to identify conflicts.
  5. Manually resolve the conflict in the file.
  6. Stage and commit the resolved changes.

Rebasing Workflow Challenge

×

Objective

Master rebasing to maintain a clean project history.

Instructions

  1. Create a new branch and make some commits.
  2. Switch back to main and make additional commits.
  3. Rebase your branch onto main using git rebase.
  4. Resolve any conflicts and complete the rebase.
  5. Ensure the history is linear using git log --oneline.

CI/CD Integration Challenge

×

Objective

Set up a basic CI/CD pipeline with GitHub Actions.

Instructions

  1. Create a new GitHub repository.
  2. Add a GitHub Actions workflow file under .github/workflows/.
  3. Define steps to run tests and build the project.
  4. Commit and push the workflow file.
  5. Trigger the workflow by pushing a code change.
  6. Monitor the workflow execution on GitHub.
